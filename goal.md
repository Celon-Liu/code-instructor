# AI Dev Progress Coach - Goal Baseline

## 项目目标

开发一个 VS Code/Cursor 插件，用于在 AI 辅助编程过程中，持续、可验证地向用户反馈：

1. 当前开发进度（不是模拟值，而是基于项目文件与状态信号）
2. 与预期目标的偏离度（低/中/高）
3. 代码真实度与有效性（避免空对空输出）
4. 下一步建议与监理告警（可执行、可验证）
5. 面向非代码用户的对话指导能力（代码指导员）

## 目标边界

- 目标只针对「当前工作区项目」生效
- 监控状态必须满足：已导入目标 + 已完成项目级评估
- 所有结论应尽量有证据来源（文件、诊断、构建、计划完成度）
- UI 以简洁清晰为主，适配浅色主题，避免强干扰视觉

## 成功标准（验收）

### A. 目标与计划基线

- 能从 `goal.md` 导入目标基线
- 能从 `plan.md` 导入执行计划
- 导入成功后 UI 明确显示状态，并保留可展开的一行目标摘要

### B. 监控与评估

- 开发进度/偏离度/监理告警为真实计算结果
- 状态卡片随代码、诊断、构建结果变更而更新
- 监控状态不再因“仅有目标”而误判为进行中

### C. 代码指导员

- 发送消息稳定可用
- 关键系统提示使用中文（在中文环境下）
- 回答包含可操作下一步，不给空泛建议

### D. 稳定性与可用性

- Webview 脚本稳定加载，按钮可点击
- 常见主机环境（VS Code/Cursor）行为一致
- 主要流程无阻断级报错

2. 保留剪贴板导入作为备用路径
3. 打通基于文件基线的评估链路（进度、偏离、告警、构建解释）
