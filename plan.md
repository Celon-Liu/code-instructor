# AI Dev Progress Coach - Execution Plan

## P0: 基线文件驱动（最高优先级）

- [ ] 新增命令：从 `goal.md` 导入目标基线
- [ ] 新增命令：从 `plan.md` 导入执行计划
- [ ] 启动时若存在这两个文件，提供“一键导入”入口
- [ ] UI 明确展示当前基线来源（goal.md/plan.md/clipboard）

## P1: 监控状态与评估联动

- [ ] 监控状态判定统一为：有目标基线 + 有项目评估结果
- [ ] 开发进度优先基于计划完成度、诊断变化、构建状态综合计算
- [ ] 偏离度映射规则固定化（绿/黄/红阈值）
- [ ] 监理告警只保留可执行、可验证语句

## P2: 构建校验与构建状态说明

- [ ] 读取配置的构建命令并执行校验
- [ ] 在 UI 中区分：空闲/运行中/通过/失败
- [ ] 失败时展示“首要修复建议”（结合 Problems/构建输出）
- [ ] 给出简短中文解释：构建校验的意义与当前状态

## P3: 代码指导员体验

- [ ] 中文环境下系统文案统一中文
- [ ] 对话输入与发送稳定，失败可见可提示
- [ ] 对话输出优先引用当前目标与计划上下文
- [ ] 保留“非代码用户友好”的说明与提问模板

## P4: UI 浅色主题优化（持续迭代）

- [x] 模块改为圆角矩形、浅边框、弱阴影
- [x] 去除深灰重框风格
- [ ] 卡片密度与留白继续微调
- [ ] 关键指标视觉层级进一步优化（状态 > 进度 > 说明）

## 回归测试清单

- [ ] 点击“导入目标”可成功导入有效目标文本
- [ ] 无效剪贴板内容时会弹输入框，不会误导入
- [ ] 导入成功后按钮与目标摘要状态正确
- [ ] 代码指导员发送可用，文案中文正常
- [ ] 监控快照与监理告警均有真实数据更新
